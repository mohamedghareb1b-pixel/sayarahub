# SayaraHub — التقرير النهائي (آخر نسخة)

## 1) التحول الجذري اللي حصل في الجلسة دي

النظام اتحول من **"مطابقة محسوبة، شخص واحد"** لـ**"بث لكل المطابقين، أول 3 ياخدوا الطلب"**، وكل مندوب بقى مستقل تماماً (مفيش تسجيل "معرض" منظّم، مفيش موافقات انضمام).

---

## 2) التسجيل (Onboarding) — 3 أسئلة بس

```
إيه اسمك؟ → إيه اسم المعرض/الجهة؟ → في أي مدينة؟
```
كل رقم جديد بياخد **"معرض شخصي"** تلقائي وراه (`isPersonalPool`) يمثل مخزونه، بدون ربط بحد تاني. بعد التسجيل يوصله 3 أزرار:
```
📤 أرسل مخزونك   📋 تفاصيل العمل   ✏️ تعديل بياناتي
```
تقدر تعدّل بياناتك في أي وقت بزرار "تعديل بياناتي".

---

## 3) المطابقة — الموديل الجديد بالكامل

**لما حد يطلب سيارة:**
1. كل مندوب عنده نفس (الماركة + الموديل + السنة + الفئة) بياخد إشعار منفصل، حتى لو أكتر من مندوب عندهم نفس العربية بالظبط.
2. كل واحد بيرد بـ**"✅ متوفر"** أو **"❌ غير متوفر"**.
3. **أول 3 يردوا بـ"متوفر"** بياخدوا الطلب فعلياً (مش واحد بس) — كل واحد فيهم بيتوصل مباشرة برقم الطالب، والطالب بياخد أرقام الـ3.
4. بعد ما يكتمل العدد (3)، أي رد متأخر من مندوب رابع بيوصله "اكتمل العدد، شكراً على ردك".

**فلو "عايز سيارة" (طلب):**
```
ماركة → موديل → فئة → سنة الصنع → لون (اختياري) → الوكيل (اختياري) → ملاحظات (اختياري)
```
- **المدينة اتشالت من الأسئلة** — بتتاخد تلقائي من بروفايل الطالب المسجل بيه.
- **السنة تقبل أكتر من قيمة** بفاصل `/` (مثال: "2024/2025") — أول سنة بتبقى الأساسية للمطابقة، والباقي بيتحط كملاحظة يشوفها المندوب ويقرر بنفسه.

**فلو "عندي سيارة" (عرض):**
- بيوريك **مخزونك الحالي الأول** قبل ما يسألك عن سيارة جديدة.
- بعدين تختار: "➕ إضافة سيارة جديدة" أو "📤 راسل الإدارة".
- ترتيب الإدخال: ماركة → موديل → فئة → سنة → **لون (إجباري، مينفعش يتخطى)** → الوكيل → ملاحظات.

---

## 4) أدوات الأدمن (كل حاجة اتبنت في الجلسات السابقة، لسه شغالة)

| الصفحة | الوظيفة |
|---|---|
| `/admin/showrooms` | تسجيل معرض/مندوب حر مسبقاً، بحث بالاسم/الرقم، عرض/تعديل/حذف مناديب |
| `/admin/inventory` | إضافة يدوية بخانات منفصلة + رفع ملف إكسل (بحث بدل قايمة طويلة) |
| `/admin/vocabulary` | تعليم البوت موديلات/ماركات/فئات/ألوان/ملاحظات جديدة، بحث + أقسام، قايمة "كلمات مستنية المراجعة" |
| `/admin/matches` | سجل كل المطابقات + أداة "🔍 مين عنده السيارة دي؟" (بحث فوري بالأرقام للنسخ) |
| `/admin/jobs` | تشغيل المهام المجدولة يدوياً |

---

## 5) الأمان

- `/admin` محمي بـ Basic Auth (`ADMIN_USERNAME`/`ADMIN_PASSWORD`).
- `/api/cron/[job]` محمي بمفتاح سري (`CRON_SECRET`).
- Middleware في المكان الصح (`src/middleware.ts`)، متوافق مع Edge Runtime.

---

## 6) نقطة مفتوحة واحدة (قرار منك)

**حالة السباق النادرة:** لو 2 مناديب دوسوا "✅ متوفر" في نفس اللحظة (فرق أجزاء من الثانية)، فيه احتمال ضئيل جداً إن الاتنين يعدّوا شرط "لسه فيه أماكن فاضية" قبل ما أي واحد يحدّث العداد — ممكن يوصل لـ4 "فائزين" بدل 3 في حالة نادرة جداً. الحل الفني (قفل قاعدة بيانات `SELECT FOR UPDATE`) متأجل لحد ما نشوف هل بيحصل فعلياً في الاستخدام الحقيقي.

---

## 7) الخطوة الجاية المنطقية

بما إن التسجيل والمطابقة اتغيروا جذرياً، **لازم اختبار كامل من الصفر** قبل أي استخدام حقيقي واسع:
1. امسح بيانات الاختبار القديمة (المعارض/المناديب اللي كانت بالنظام القديم).
2. سجّل 3-4 مناديب تجريبيين جدد بالفلو الجديد.
3. جرب سيناريو كامل: مندوب يضيف سيارة → مندوب تاني يطلب نفس السيارة → تأكد إن الإشعار وصل، وإن "✅ متوفر" وصّلهم صح، وإن ردود متأخرة من مناديب زيادة اتقفلت صح.
